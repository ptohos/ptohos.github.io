Compiling και linking υποπρογραμμάτων

Υποπρογράμματα μπορούν να γραφούν σε διαφορετικά files από αυτό του κυρίου προγράμματος. Έτσι μπορείτε να τα χρησιμοποιήσετε σε διαφορετικές περιπτώσεις κυρίων προγραμμάτων. Στην περίπτωση αυτή θα πρέπει να τα κάνετε ωστόσο compile και link μαζί με το κύριο πρόγραμμα. Για παράδειγμα έστω ότι το κύριο πρόγραμμά σας βρίσκεται στο file main.f. και το υποπρόγραμμα που καλείται βρίσκεται σε ένα διαφορετικό file με όνομα submain.f. Τότε για να τα κάνετε compile ώστε το executable πρόγραμμα να ξέρει που θα βρει το υποπρόγραμμα θα γράψετε την εντολή:  gfortran -o main.x  main.f submain.f   Δίνετε δηλαδή τα ονόματα και των 2 files.

Αν θέλατε να δείτε ότι το υποπρόγραμμα που γράψατε στο δικό του file (submain.f) δεν έχει "γραμματικά λάθη" (λάθη γλώσσας δηλαδή) θα πρέπει να χρησιμοποιήσετε την εντολή gfortran ως εξής:  gfortran -c submain.f   Η επιλογή "-c" κάνει compilation και linking χωρίς ωστόσο να προσπαθεί να δημιουργήσει executable πρόγραμμα (.x) που για υποπρογράμματα δεν έχουν νόημα. Η παραπάνω εντολή θα σας δώσει ένα file με το όνομα submain.o (object file). Έτσι αργότερα όταν κάνετε compile, linking και building του executable προγράμματός σας θα μπορούσατε να γράψετε gfortran -o main.x main.f submain.o Στην περίπτωση αυτή δεν χρειάζεται να κάνει compilation του submain.f μια και ήδη υπάρχει. Απλά κάνει compilation του main.f και όταν φθάνει στο στάδιο όπου κάνει build το executable θα χρησιμοποιήσει και το object file του submain.o.

Δημιουργία προσωπικής βιβλιοθήκης υποπρογραμμάτων

Οι διάφορες βιβλιοθήκες αποτελούνται από συλλογή των object files των διαφόρων συναρτήσεων. Επομένως αν θελήσετε να δημιουργήσετε τη δική σας βιβλιοθήκη συναρτήσεων ή υποπρογραμμάτων τότε θα πρέπει να μετατρέψετε το  κώδικα κάθε υποπρογράμματος σε object file (όπως αναφέρθηκε παραπάνω με την εντολή  gfortran -c submain.f). Αφού έχετε μια λίστα από object files, μπορείτε να δημιουργήσετε μια βιβλιοθήκη με το κάποιο όνομα, έστω libmylib.a χρησιμοποιώντας την ακόλουθη εντολή: ar -r libmylib.a submain1.o submain2.o submain3.o  κ.ο.κ. Η εντολή ar δημιουργεί τη βιβλιοθήκη libmylib.a (Προσοχή: το όνομα της βιβλιοθήκες ξεκινά από lib και έχει ακρωνύμιο .a) η οποία αποτελείται από 3 object files (submain1.o, submain2.o, submain3.o). Για να δείτε τα περιεχόμενα μιας βιβλιοθήκης δώστε την εντολή: ar -t mylib.lib. H εντολή ar θα μπορούσε πέρα από τις επιλογές (-r: replace, -t: list) να έχει επίσης -q: quick append (δηλαδή προσθήκη νέου file στη βιβλιοθήκη κρατώντας τα υπάρχοντα) και -d: delete (συγκεκριμένο file από τη βιβλιοθήκη).

Για να κάνετε link τη βιβλιοθήκη σας με κάποιο πρόγραμμα τότε μπορείτε να δώσετε την εντολή gfortran -o main.x main.f -llibmylib.a (ένα επιπλέον l στην αρχή) ή διαφορετικά gfortran -o main.x main.f -lmylib   Στην περίπτωση αυτή το σύμβολο "-l" (προσοχή el) δηλώνει ότι το όνομα μιας Unix βιβλιοθήκης libmylib.a
