Οδηγίες για τη χρήση της εντολής tar

 

Η εντολή tar επιτρέπει τη αποθήκευση πολλών αρχείων σε ένα και μόνο νέο αρχείο ενώ παράλληλα ελαττώνει το χώρο που απαιτείται για φύλαξη διαφόρων αρχείων. Δύο από τους πιο χρήσιμες εφαρμογές της εντολής είναι η δημιουργία αρχείου και η επαναφορά αρχείων από ένα τέτοιο tar αρχείο στην κανονική τους μορφή.


Δημιουργία tar file:  

Πληκτρολογήστε την εντολή:   tar  -czvf   <filename>.tgz   file1.f   file2.f   file3.f

 

Θα μπορούσατε αντί να χρησιμοποιήσετε το όνομα του κάθε αρχείου ξεχωριστά να αντικαταστήσετε τα ονόματα όλων των αρχείων που έχουν την ίδια προέκταση (π.χ. .f) με τον wild χαρακτήρα (*). Για παράδειγμα η πιο πάνω εντολή μπορεί να γραφεί:

 tar -czvf  <filename>.tgz *.f  *.dat  *.txt

Όλα τα αρχεία με προέκταση .f, *.dat και *.txt    (file1, file2, file3) που βρίσκονται στο φάκελο που βρίσκεστε θα αρχειοθετηθούν στο tar αρχείο.

Θα μπορούσατε να κάνετε tar ένα ολόκληρο directory καθώς και τους  subdirectories που υπάρχουν μέσα σε αυτόν δίνοντας την εντολή tar -czvf <filename>.tgz  <directory name>/

 

Εξαγωγή αρχείων από tgz file:

Για να εξάγετε τα files από ένα tgz file δώστε την εντολή: tar -xzvf <flename>.tgz Αυτή η εντολή θα εξάγει όλα τα files ή directories που βρίσκονται μέσα στο file και θα αποθηκεύσει στο directory που βρίσκεστε. Θα μπορούσατε να βγάλετε μόνο κάποια συγκεκριμένα files αν ξέρετε τα ονόματά τους. Για παράδειγμα tar -xzvf <filename>.tgz file1.f θα εξάγει το file file1.f που βρίσκεται στο tgz file.


Περιεχόμενα ενός tgz file:

Για να δείτε το περιεχόμενο ενός tgz file δώστε την εντολή tar -tzvf <filename>.tgz 

 

Για περισσότερες πληροφορίες σχετικά με την εντολή tar και το τι σημαίνουν τα γράμματα (c, x, v, f) που χρησιμοποιήσαμε παραπάνω μπορείτε να πληκτρολογήσετε την εντολή man tar  (man είναι η αντίστοιχη εντολή help σε περιβάλλον Linux/Unix)

Οδηγίες για την μεταφορά αρχείων μεταξύ υπολογιστών

Για την μεταφορά αρχείων μεταξύ υπολογιστών του εργαστηρίου πληροφορικής και εργαστηρίου φυσικής (IP διεύθυνση: estia.phy.ucy.ac.cy)

  • Ανοίξτε ένα παράθυρο terminal και πληκτρολογήστε την εντολή:
    scp  filename (που θέλετε να μεταφέρετε)   <username>@estia.phy.ucy.ac.cy:~/.

  • Η εντολή αυτή θα πάρει το το filename που πληκτρολογήσατε και θα το μεταφέρει στο home directory σας στους υπολογιστές της πληροφορικής. Προσέξτε ότι μετά την ηλεκτρονική διεύθυνση υπάρχει (:) και στο μετά το (/) υπάρχει (.) που σημαίνει ότι το file θα έχει το ίδιο όνομα. Τα (<,>) δεν χρειάζονται. Απλά δηλώνουν την επιλογή του ονόματος.

  • Για να μεταφέρετε ένα ολόκληρο directory και subdirectory tree κάντε το ακόλουθο:
     scp -r <dir>  <username>@estia.phy.ucy.ac.cy:~/.
    Για παράδειγμα:  scp  -r Lab07  fotis@estia.phy.ucy.ac.cy:~/.  θα μεταφέρει όλα τα files του directory Lab07 συμπεριλαμβανομένων και των τυχόν υπο-directories σε ένα άλλο μηχάνημα και θα δημιουργήσει τον ίδιο directory (Lab07) και όλους τους υπό-directories.

  • Σημειωτέον ότι όταν πατήσετε <enter> θα σας ρωτήσει αν θέλετε να σώσετε  την ηλεκτρονική διεύθυνση που θέσατε. Απαντήστε yes. Κατόπιν θα σας ρωτήσει το  password που αντιστοιχεί στο username που θέσατε. Πληκτρολογήστε το password που έχετε.

  • Οι υπολογιστές του εργαστηρίου της πληροφορικής (γενική ΙP διεύθυνση b103ws<number>.in.cs.ucy.ac.cy) δεν είναι προσβάσιμοι έξω από το εργαστήριο εκτός και αν έχετε account στο VPN της πληροφορικής. Επομένως δεν μπορείτε να μεταφέρετε αρχεία στους υπολογιστές της πληροφορικής από κάποιο άλλο υπολογιστή ή το σπίτι σας. Για να αποκτήσετε το λογισμικό VPN ακολουθήστε αυτό το link. Όλα τα νούμερα των υπολογιστών της πληροφορικής τα οποία πρέπει να δώσετε στην IP διεύθυνση περιέχονται μεταξύ 1 - 33

Η εντολή scp είναι η πλέον ενδεδειγμένη γιατί κάνει ασφαλή αντιγραφή (Secure CoPy scp). Δηλαδή το password δεν μπορεί να "κλαπεί" κατά την πληκτρολόγηση.

Πως μπορείτε να ανοίξετε window σε οθόνη που βλέπετε που δεν είναι συνδεδεμένη στον υπολογιστή που δουλεύετε

  • Συχνά χρειάζεται να κάνουμε login σε ένα διαφορετικό υπολογιστή (remote computer) από αυτό στον οποίο καθόμαστε. Στην περίπτωση αυτή θέλουμε να μεταφέρουμε τα παράθυρα που ανοίγουμε στο remote computer (π.χ. το παράθυρο του emacs, του gnuplot) στην οθόνη του υπολογιστή που καθόμαστε. (Το πρόβλημα αυτό το έχουμε συναντήσει συχνά στα εργαστήρια της πληροφορικής με υπολογιστές που δεν μπορούν να κάνουν compile οποιοδήποτε κώδικα - βλέπουμε ένα μήνυμα της μορφής:  ΚΟF something).  
  • Αυτό που χρειάζεται να κάνουμε,  είναι να μεταφέρουμε τη μεταβλητή περιβάλλοντος DISPLAY (environment variable) από το remote computer στον υπολογιστή που καθόμαστε, δηλαδή σε εκείνον που είναι συνδεδεμένη η οθόνη (DISPLAY) έχουμε μπροστά μας.
  • Η εντολή που δίνουμε εξαρτάται από το φλοιό (shell) εντολών στο οποίο δουλεύουμε. Στους υπολογιστές του εργαστηρίου της πληροφορικής δουλεύουμε στο  tcsh-sell ενώ στους υπολογιστές του εργαστηρίου της φυσικής δουλεύουμε σε bash-shell.
  • Για το tcsh-sell η εντολή μεταφοράς του DISPLAY είναι:    
                              setenv      DISPLAY  <local_computer_address>:0.0
    όπου local_computer_address είναι η ηλεκτρονική διεύθυνση του computer στον οποίο είναι συνδεδεμένη η οθόνη που θέλουμε να χρησιμοποιήσουμε. Για παράδειγμα αν έχουμε κάνει login στον υπολογιστή cs<number1>.in.cs.ucy.ac.cy (εν γένει μπορούμε να κάνουμε login σε άλλο υπολογιστή από αυτό που χρησιμοποιούμε με την εντολή:  ssh b103ws<number2>.in.cs.ucy.ac.cy) ενώ καθόμαστε στον υπολογιστή με διεύθυνση cs<number2>.in.cs.ucy.ac.cy) και θέλουμε να μεταφέρουμε το DISPLAY του 2 στον 1, θα πληκτρολογήσουμε: setenv DISPLAY b103ws<number2>.in.cs.ucy.ac.cy:0.0.  Σημειώστε ότι η λέξη DISPLAY γράφεται με κεφαλαίους χαρακτήρες, ενώ μετά την διεύθυνση βάζουμε :0.0.
  • Για το bash-shell η εντολή μεταφοράς του DISPLAY είναι: 
                              export DISPLAY=<local_computer_address>:0.0

Οδηγίες για να καθαρίσετε το home directory σας από files που δημιουργεί ο browser mozilla

Πολύ συχνά εμφανίζεται το φαινόμενο να μη μπορείτε να σώσετε κάποιο file το οποίο γράφετε γιατί ο χώρος του δίσκου που σας έχει δοθεί έχει γεμίσει (το μήνυμα που σας δίνει ο υπολογιστής είναι ότι disk quota exceeded) ή να μην μπορείτε να κάνετε login (και στην περίπτωση αυτή η αιτία είναι ότι έχει γεμίσει ο χώρος του δίσκου που σας δίνεται είναι γεμάτος).
Χρειάζεται και στις δύο περιπτώσεις να σβήσετε κάποια files που βρίσκονται στο directory .mozilla/firefox (προσέξτε τη τελεία εμπρός από το όνομα mozilla).
 
Αν το πρόβλημα εμφανιστεί ενώ είστε logged in σε κάποιο PC του εργαστηρίου της πληροφορικής τότε κάνετε τα ακόλουθα:

(1) cd ~/.mozilla/firefox
(2) ls -al   Θα πρέπει να δείτε δυο files (directories) το ένα εκ των οποίων έχει extension <name>.default. Το <name> είναι διαφορετικό για κάθε user.
(3) cd <name>.default   Πηγαίνετε στο subdirectory.
(4) ls -altr   Θα πρέπει να δείτε ένα file το όνομα του οποίου είναι urlclassifier3.sqlite και το μέγεθός του είναι περίπου 23ΜΒ.
(5) Σβήστε το file αυτό:   rm urlclassifier3.sqlite
(6) Μπορείτε να σβήσετε επίσης και τα files:  XPC.mfasl,  XUL.mfasl, και places.sqlite. Όλα τα files βρίσκονται στο ίδιο subdirectory.
Είναι επίσης καλό να κάνετε empty trash πιέζοντας το εικονίδιο του trash can στο desktop. Αυτό γιατί αν έχετε σβήσει κάποια files μεταφέροντάς τα στο trash can, τα files αυτά εξακολουθούν να καταλαμβάνουν χώρο του προσωπικού σας δίσκου. Έχουν απλά μεταφερθεί σε διαφορετικό directory. Επομένως κάνοντας empty trash τα σβήνετε τελείως.

Αν το πρόβλημα εμφανίζεται όταν προσπαθήσετε να κάνετε logging.Επειδή ο window manager χρειάζεται να δημιουργήσει log files όταν ξεκινά, δεν βρίσκει τον απαραίτητο χώρο και επομένως δεν σας αφήνει να μπείτε. Χρειάζεται να κάνετε login χρησιμοποιώντας το terminal ενός συναδέλφου σας.
(1) Ανοίξτε το terminal window από το session ενός συναδέλφου σας που είναι logged in.
(2) Πληκτρολογήστε στο terminal την εντολή:
ssh <username>@b103ws10.in.cs.ucy.ac.cy   όπου <username> είναι ο κωδικός χρήστη. Θα σας ρωτήσει ίσως αν θέλετε να κρατήσετε το όνομα του υπολογιστή που πληκτρολογήσατε (b103ws10). Πληκτρολογήστε yes. Κατόπιν σας ζητά το password. Πληκτρολογήστε το password σας. Προσέξτε ότι ενώ πληκτρολογείτε το password δεν εμφανίζεται τίποτα στην οθόνη (ούτε οι τελείες που εμφανίζονται όταν κάνετε κανονικό loggin στο PC).
(3) Πατήστε return στο τέλος του password.  Αν δώσατε το σωστό password θα έχετε μπεί στο λογαριασμό σας. Αν όχι θα σας ζητήσει και πάλι το password.
(4) Επαναλάβετε τα βήματα (1)-(6) που περιγράφηκαν πιο πάνω.
Στο τέλος της διαδικασίας πληκτρολογήστε exit για να κάνετε logout από το terminal
