ΕΞΑΣΚΗΣΗ

  • Κάνετε logon στο σύστημα. Δημιουργήστε ένα καινούριο directory στο home directory σας και ονομάστε τον LAB02. Θα πρέπει να πάτε στο νέο αυτό directory και ότι προγράμματα γράψετε στο σημερινό εργαστήριο θα πρέπει να βρίσκονται σε αυτό τον directory.

  • Για να κάνετε compile ένα πρόγραμμα ονομάστε το αρχείο που περιέχει το πρόγραμμα σας σαν <name>.f, όπου <name> ότι όνομα θέλετε.  Δηλαδή είναι fortran file. Κατόπιν έξω από το emacs εκτελέστε  την ακόλουθη εντολή gfortran <name>.f  -o  <name>.x (ανάλογα με τον compiler που είναι εγκατεστημένος στο λειτουργικό σύστημα η εντολή μπορεί να χρειαστεί να δώσετε την εντολή g77 αντί για gfortran). Με την εντολή αυτή κάνετε compile το πρόγραμμα και παράλληλα φτιάχνετε ένα executable (<name>.x) το οποίο μπορείτε να τρέξετε και να δείτε το αποτέλεσμα του προγράμματός σας. Για να το "τρέξετε" πληκτρολογήστε <name>.x  ή ./<name>.x. Aν δεν είχαμε ορίσει το -ο <name>.x τότε το αρχείο που θα γράφονταν θα ήταν <name>.out.

Κάνετε compilation (η εντολή που θα πρέπει να πληκτρολογήσετε είναι: g77 -o constvar.x constvar.f) και τρέξτε το executable που θα πάρετε ( ./constvar.x )
     

  • Με βάση το προηγούμενο πρόγραμμα, γράψτε τώρα ένα δικό σας το οποίο δέχεται  4 πραγματικούς αριθμούς (a,b,c,d) και 2 ακέραιους (i,j) από το πληκτρολόγιο και κατόπιν τυπώνει στην οθόνη τα αποτελέσματα των ακόλουθων πράξεων: (α) z = (a+b)/(c+d)   (b) z = a+b/(c+d)   (c) z = (a+b)/c + d, (d) k = i/j και (e) z = 4**(i/j).  Δοκιμάστε να δώσετε τις ακόλουθες τιμές:  α = 10, β = 3.4, c = 12, d = 2, i = 1 και j = 2.
    Νομίζετε ότι το αποτέλεσμα των δυο τελευταίων πράξεων (d) και (e) (μεταξύ των ακεραίων) είναι σωστό; Καταλαβαίνετε τι πρόβλημα υπάρχει;
